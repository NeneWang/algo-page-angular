Bubble Sort

Análisis de Complejidad del Bubble Sort

Mejor Caso Caso Promedio Peor Caso
O(n) O(n) O(n^2)

Peor Caso:

En el peor caso, Bubble Sort debe realizar n - 1 comparaciones y n - 1 intercambios, donde "n" es el número de elementos en la lista. Esto se debe a que el algoritmo compara y posiblemente intercambia cada par de elementos en la lista en cada pasada. En cada pasada, el elemento más grande "burbujea" hacia la parte derecha de la lista.

La cantidad total de comparaciones e intercambios necesarios para ordenar la lista de "n" elementos en el peor caso se puede expresar como:

  • Comparaciones: (n - 1) + (n - 2) + ... + 1 = (n * (n - 1)) / 2
  • Intercambios: (n - 1)

Por lo tanto, en el peor caso, la complejidad de tiempo de Bubble Sort es O(n^2), ya que el término dominante es n^2.

Mejor Caso:

El mejor caso ocurre cuando la lista ya está ordenada. En este caso, Bubble Sort solo necesita hacer un pase a través de la lista para verificar que no se realicen intercambios. El número de comparaciones será igual a n - 1, pero no se realizarán intercambios. Por lo tanto, en el mejor caso, la complejidad de tiempo es O(n).

Caso Promedio:

El caso promedio es difícil de determinar con precisión, ya que depende de la distribución de los elementos en la lista. Sin embargo, en promedio, Bubble Sort sigue realizando O(n^2) comparaciones e intercambios. Esto lo hace ineficiente para listas grandes.

En resumen, Bubble Sort es un algoritmo de ordenamiento ineficiente en términos de tiempo, especialmente para listas grandes, debido a su complejidad cuadrática. Hay algoritmos de ordenamiento más eficientes, como Quick Sort, Merge Sort y Heap Sort, que suelen ser preferibles en la mayoría de las situaciones. Bubble Sort se utiliza principalmente con fines educativos o cuando se tiene una cantidad muy pequeña de datos para ordenar.
